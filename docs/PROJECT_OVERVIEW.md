# Project Overview — Motorcycle Dealership ERP (Laos MVP)

## ภาพรวมโครงการ

**Motorcycle Dealership ERP** คือระบบจัดการร้านจำหน่ายรถจักรยานยนต์แบบ MVP (Minimum Viable Product) ที่ออกแบบมาเฉพาะสำหรับตลาดลาว โดยเน้นฟีเจอร์ที่จำเป็นจริง แก้ Pain points ของร้านมอเตอร์ไซค์ได้ตรงจุด และรองรับวิธีทำธุรกิจที่เป็นเอกลักษณ์ในประเทศลาว — ตั้งแต่การติดตามรถแต่ละคันที่มีเลขถัง/เลขเครื่องไม่ซ้ำกัน ไปจนถึงการรับชำระผ่าน Mobile Banking และการผ่อนชำระหลายรูปแบบ

โปรเจกต์นี้สร้างบนพื้นฐาน **Modular Monolith ERP** (Bun + React + Elysia + PostgreSQL) ที่มีระบบ Auth, RBAC, Audit Log และ UI ภาษาลาวพร้อมใช้งานแล้ว โดยจะต่อยอดเป็นโมดูลธุรกิจตาม MVP ด้านล่าง

---

## ปัญหาที่ระบบแก้ไข (Pain Points)

| ปัญหาในตลาดลาว | วิธีที่ระบบแก้ |
|---|---|
| รถมอเตอร์ไซค์ไม่ใช่สินค้าทั่วไป — แต่ละคันมีเลขถัง/เลขเครื่อง/แบตเตอรี่เฉพาะตัว | ระบบติดตามรถรายคัน (Unit-level tracking) |
| เอกสารนำเข้า/ทะเบียนป้ายเหลือง-ป้ายขาวซับซ้อน | ติดตามสถานะเอกสารว่าครบหรือพร้อมทำทะเบียนแล้วหรือยัง |
| ลูกค้าชอบซื้อสดและผ่อน ทั้งผ่านไฟแนนซ์และร้านปล่อยเอง | รองรับ Hybrid Financing 2 รูปแบบ |
| ราคานำเข้าผันผวนตามอัตราแลกเปลี่ยน | Multi-Currency (LAK / THB / USD) |
| ลูกค้าโอนเงินผ่าน Mobile Banking เป็นหลัก | อัปโหลดสลิป + แยกบัญชีตามช่องทางรับเงิน |
| EV มีการรับประกันแบตเตอรี่แยกจากตัวรถ | Warranty Tracking แยกตามประเภท |

## ขอบเขต MVP — 4 โมดูลหลัก

### 1. ระบบจัดการสต็อกและติดตามรถ (Inventory & Vehicle Tracking)

รถจักรยานยนต์มีความเฉพาะตัวสูงกว่าสินค้าทั่วไป ระบบจึงออกแบบให้ติดตาม **รายคัน (Unit)** ไม่ใช่แค่รุ่น/จำนวน

**ฟีเจอร์หลัก:**

- **Chassis & Engine Tracking** — บันทึกเลขถัง (Chassis Number) และเลขเครื่อง (Engine Number) สำหรับรถน้ำมัน
- **EV Battery Mapping** — สำหรับรถไฟฟ้า: เลขแบตเตอรี่ (Battery Serial Number), ความจุ และข้อมูลที่เกี่ยวกับการรับประกันแยกจากตัวรถ
- **สถานะเอกสาร** — ติดตามเอกสารนำเข้า (ใบเสียภาษี, ใบตรวจทางเทคนิค) ว่าครบถ้วนหรือพร้อมทำทะเบียนป้ายเหลือง/ป้ายขาวแล้วหรือไม่

---

### 2. ระบบการขายและเงินผ่อน (Sales & Financing Options)

ตลาดลาวมีทั้งการซื้อสดและการผ่อนหลายรูปแบบ ระบบรองรับทั้งสองแนวทาง

**Hybrid Financing — 2 รูปแบบ:**

| รูปแบบ | รายละเอียด |
|---|---|
| **ผ่านสถาบันการเงิน (Finance/Bank)** | บันทึกบริษัทไฟแนนซ์ (เช่น Welcome, วัฒนา, T-Ke) และติดตามเงินโอนเข้าจากไฟแนนซ์ |
| **ร้านปล่อยผ่อนเอง (In-House Leasing)** | คำนวณค่างวด, ดอกเบี้ย, วันครบกำหนดชำระ พร้อมตารางผ่อนชำระ (Payment Schedule) |

**Multi-Currency Engine:**

- ตั้งราคาและรับชำระเป็น **กีบ (LAK)**, **บาท (THB)** และ **ดอลลาร์ (USD)**
- รองรับความผันผวนของอัตราแลกเปลี่ยนในการนำเข้ารถ

---

### 3. ระบบรับชำระเงินและตรวจสอบสลิป (Payment & Slip Verification)

สะท้อนพฤติกรรมการชำระเงินผ่าน Mobile Banking ที่นิยมในลาว

**ฟีเจอร์หลัก:**

- **Slip Upload & QR Code** — หน้ารับชำระให้พนักงานขายอัปโหลดสลิปโอนเงินจากลูกค้า (BCEL One หรือธนาคารอื่น) เป็นหลักฐาน
- **บัญชีแยกตามช่องทาง** — แยกยอดเงินเข้าระหว่าง:
  - เงินสดหน้าร้าน
  - บัญชีธนาคาร A
  - บัญชีธนาคาร B

  เพื่อให้เจ้าของร้าน Reconcile ยอดเงินท้ายวันได้ง่าย

---

### 4. ระบบบริการหลังการขายและประกัน (After-Sales & Warranty)

**CRM & เอกสารลูกค้า:**

- ประวัติลูกค้า, เบอร์โทร, ที่อยู่ (บ้าน / เมือง / แขวง)
- ข้อมูลบัตรประชาชน/ทะเบียนบ้าน (สำหรับเตรียมเอกสารทะเบียนป้าย)

**Warranty Tracking (เน้น EV):**

- แยกการแจ้งเตือนตามประเภท:
  - ประกันตัวรถ
  - ประกันมอเตอร์
  - ประกันแบตเตอรี่ (EV)

**Service Check-in (ระยะ MVP):**

- บันทึกการเข้าเปลี่ยนน้ำมันเครื่อง
- ตรวจสอบสภาพแบตเตอรี่/ระบบไฟฟ้า ตามระยะทาง

## สถาปัตยกรรมทางเทคนิค

```
┌─────────────────────────────────────────────────────────┐
│                    Frontend (React 19)                   │
│         TanStack Router │ TanStack Query │ PWA           │
├─────────────────────────────────────────────────────────┤
│              Modular Monolith (Feature Modules)          │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐   │
│  │ Inventory│ │  Sales   │ │ Payment  │ │After-Sales│   │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘   │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐   │
│  │   Auth   │ │  Users   │ │   RBAC   │ │  Audit   │   │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘   │
├─────────────────────────────────────────────────────────┤
│              Backend (ElysiaJS + Drizzle ORM)            │
│                    PostgreSQL Database                   │
└─────────────────────────────────────────────────────────┘
```

| ชั้น | เทคโนโลยี |
|---|---|
| Runtime | Bun |
| Frontend | React 19, TypeScript, Tailwind CSS |
| Routing / State | TanStack Router, TanStack Query |
| Backend | ElysiaJS |
| Database | PostgreSQL + Drizzle ORM |
| Auth | Better Auth (JWT Sessions) |
| Authorization | RBAC แบบละเอียด |
| Compliance | Audit Logging |
| UX | PWA, UI ภาษาลาว, Responsive |

## สถานะปัจจุบันของโปรเจกต์

**พร้อมใช้งานแล้ว (Foundation):**

- Authentication & Session Management
- User Management
- RBAC (Roles & Permissions)
- Audit Logging
- Dashboard
- UI ภาษาลาว + PWA

**อยู่ในแผน MVP (ยังไม่ได้ implement):**

- Inventory & Vehicle Tracking
- Sales & Financing
- Payment & Slip Verification

## เป้าหมาย MVP

1. **แก้ Pain points ได้ตรงจุด** — ไม่ใส่ฟีเจอร์เกินความจำเป็น
2. **เหมาะกับตลาดลาว** — Mobile Banking, Multi-Currency, เอกสารทะเบียน, การผ่อนหลายรูปแบบ
3. **รองรับ EV** — Battery Mapping และ Warranty แยกประเภท
4. **ขยายต่อได้** — Modular Monolith แยกโมดูลชัดเจน สามารถเพิ่มฟีเจอร์หรือแยกเป็น Microservices ในอนาคตได้
